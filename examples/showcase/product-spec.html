Product Spec

Live reload for HTML artifacts

Replace manual browser refreshes with an instant, watcher-driven preview for AI-generated HTML files. Ships with v1.

TL;DR

When a file in the open workspace changes on disk, the preview re-renders within 300 ms. No localhost. No build step. Works for any .html regardless of which agent generated it.

Problem

Today, users with AI-generated HTML loop through: open in browser, see a bug or missed detail, switch back to the chat, ask the agent to fix, wait for the file to be rewritten, switch back to the browser, hit refresh. That cycle costs ~6 context switches per iteration.

Goals

MetricTarget
File save → preview update< 300 ms p99
Memory overhead per workspace< 50 MB
CPU at idle0%
Setup steps from "open folder" to "preview"1

Requirements

  • Must
    Watcher debounces rapid consecutive saves (300 ms) so a streaming agent doesn't trigger 50 re-renders per second.
  • Must
    Preview state (scroll position, internal anchor) is preserved across reloads when possible.
  • Must
    Sandbox: no JS / network in Safe mode by default; explicit Trusted opt-in per workspace.
  • Should
    Sidebar reflects new files added by the agent without a re-scan.
  • Could
    Visual diff highlighting the changed region for ~500 ms post-update.

Out of scope (v1)

  • Multi-file dependency tracking (e.g. CSS in a sibling file)
  • HMR for partial DOM updates — full reload is fine for v1
  • Cross-window broadcasting if user opens two workspaces
